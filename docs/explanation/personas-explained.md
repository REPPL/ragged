### The Three Personas

#### 1. The Researcher
- **Who**: Academic, professional researcher
- **Needs**: Precision, citations, methodology
- **Use Case**: Literature reviews, research papers
- **Interface**: CLI, detailed outputs, evaluation metrics

#### 2. The Professional
- **Who**: Knowledge worker, developer
- **Needs**: Quick answers, context, reliable sources
- **Use Case**: Documentation, code exploration, work documents
- **Interface**: Web UI, balanced detail, fast responses

#### 3. The Casual User
- **Who**: Personal notes, learning, exploration
- **Needs**: Simple interface, conversational, accessible
- **Use Case**: Personal documents, journals, learning materials
- **Interface**: Web UI, minimal configuration, friendly

### Progressive Disclosure
- How personas guide feature visibility
- Simple by default, powerful when needed
- Expert modes and advanced features
- Customisation and preferences

### Design Implications
- Interface complexity management
- Feature discovery and onboarding
- Documentation structure

### Real-World Scenarios
- Research assistant mode
- Code documentation explorer

- Learning companion

### Balancing Needs
- Avoiding feature bloat
- Maintaining usability
- Power user satisfaction
- Beginner friendliness
