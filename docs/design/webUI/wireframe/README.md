# Web UI Wireframes

Wireframe documentation for ragged's web user interface.

## Purpose

Low-fidelity wireframes showing layout, structure, and user flow for the ragged web interface.

## Contents

Wireframe files showing:
- Page layouts
- Component structure
- User interaction flows
- Navigation patterns

## What Belongs Here

- Low-fidelity wireframes (.svg, .penpot, .sketch, etc.)
- User flow diagrams
- Layout specifications
- Component placement mockups
- Navigation structure diagrams

## What Doesn't Belong Here

- **High-fidelity mockups** → See `../`
- **Icon assets** → See `../icons/`
- **Implementation code** → See `src/web/`
- **Architecture decisions** → See `../../development/decisions/adrs/`

## Wireframe Standards

Wireframes should:
1. Focus on layout and structure, not visual design
2. Show component hierarchy clearly
3. Indicate user interaction points
4. Document responsive behaviour
5. Include accessibility considerations
