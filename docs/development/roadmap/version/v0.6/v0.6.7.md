# v0.6.7: Svelte Core UI

**Focus:** Foundation SvelteKit application with core document and query interfaces

**Hours:** 25-35 hours

**Dependencies:** v0.6.6 (FastAPI backend)

**Status:** Planned

---

## Overview

Version 0.6.7 creates the foundational Svelte/SvelteKit web application to replace the Gradio demo UI from v0.5.4. This version establishes the core application structure, routing, and primary user interfaces for document management and queries.

**What This Version Delivers:**
- SvelteKit application setup and architecture
- Document upload and management interfaces
- Multi-modal query interfaces (text, image, hybrid)
- Results display with basic formatting
- Core component library
- Tailwind CSS styling foundation

---

## Key Features

### Application Foundation
- SvelteKit SSR + SPA routing
- TypeScript configuration
- Tailwind CSS setup
- API client integration with v0.6.6 FastAPI
- Layout and navigation structure

### Document Management
- Drag-and-drop PDF upload interface
- Batch upload with progress tracking
- Document library with search/filter
- Basic document preview
- Document metadata display

### Multi-Modal Queries
- Text query input with autocomplete
- Image query upload with preview
- Hybrid query interface (text + image)
- Weight sliders for hybrid queries
- Query history display

### Results Display
- Search results list
- Retrieved chunks display
- Source document references
- Relevance scoring display

## UI Mockup Guidance

### Pre-Implementation Design

**Purpose**: Create UI mockups before coding to:
- Validate component structure
- Identify reusable patterns
- Get early stakeholder feedback
- Reduce rework during implementation

### Recommended Tools

**Primary: Penpot** (Open Source)
- Free, no limits
- Figma-like interface
- Web-based: https://penpot.app or self-hosted
- SVG-based, future-proof exports
- **Best for**: Detailed component design, layouts, interactive prototypes

**Secondary: Excalidraw** (Open Source, Quick Sketches)
- Fast wireframing
- Hand-drawn aesthetic
- Great for early-stage iteration
- **Best for**: Rough wireframes, quick concept validation

**Alternative: Figma Free Tier**
- 3 files limit (sufficient for this project)
- Full features, professional ecosystem
- **Best for**: If already familiar with Figma

### Core Components to Mock

**Priority 1: Essential Components (Week 1)**
- Layout shell (header, sidebar, main area)
- Document upload interface (drag-and-drop zone)
- Query input components (text, image, hybrid)
- Results list display
- Navigation structure

**Priority 2: Supporting Components (Week 2)**
- Document library grid/list view
- Document preview modal
- Query history sidebar
- Loading states and progress indicators
- Error states and empty states

**Priority 3: Polish (Week 3)**
- Component hover/active states
- Responsive breakpoints (mobile, tablet, desktop)
- Theme variations (light/dark mode previews)

### Mockup Deliverables

**Before v0.6.7 implementation starts:**
- [ ] Layout mockups (desktop and mobile)
- [ ] Component library mockup (buttons, inputs, cards)
- [ ] Key user flows (upload → query → results)
- [ ] Responsive breakpoints defined
- [ ] Color palette and typography documented

**Storage**: Store mockups in `/docs/design/ui/v0.6.7/` (export as SVG/PNG from Penpot)

### Design System References

**Inspiration** (for production-quality RAG UI):
- Perplexity.ai (query interface, results display)
- Notion (document management, clean layouts)
- Linear (keyboard shortcuts, command palette)
- Excalidraw (minimalist, hand-drawn friendly aesthetic)

**Component Libraries** (to consider for Svelte):

- Skeleton UI (Svelte-specific, comprehensive)
