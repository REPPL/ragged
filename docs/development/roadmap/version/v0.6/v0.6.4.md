## Overview

Version 0.6.4 introduces streaming response generation, parallel retrieval processing, and CLI analytics commands to improve both perceived and actual performance.

**What This Version Delivers:**
- OPTIMISE-007: CLI Analytics Commands (15-20h)
- OPTIMISE-008: Streaming Response Generation (12-15h)
- OPTIMISE-009: Parallel Retrieval Pipeline (15-20h)

## Features

### OPTIMISE-007: CLI Analytics Commands (15-20 hours)

**New CLI Commands:**
- `ragged analytics` - Performance metrics
- `ragged analytics routing` - Model routing stats
- `ragged analytics cache` - Cache performance
- `ragged analytics domain` - Domain adaptation stats
- `ragged analytics context` - Context scope usage

---

### OPTIMISE-008: Streaming Response Generation (12-15 hours)

**Benefits:**
- Time to first token: <1 second
- Perceived latency reduction: 60-80%
- Immediate user feedback
- Progress indication

**Success Criteria:**
- [ ] Token streaming functional
- [ ] Time to first token <1s
- [ ] CLI displays progressive output
- [ ] Gradio shows real-time generation

---

### OPTIMISE-009: Parallel Retrieval Pipeline (15-20 hours)

**Performance Gains:**
- Sequential: 450ms (vector 200ms + BM25 150ms + rerank 100ms)
- Parallel: 300ms (max(vector, BM25) + rerank = 200ms + 100ms)
- **Reduction:** 33-60%

**Success Criteria:**
- [ ] 40-60% retrieval latency reduction
- [ ] No accuracy regression
- [ ] Concurrent operations synchronized
