## Overview

Version 0.6.1 implements automatic model routing based on query characteristics from v0.6.0's classification system. Different model sizes and specialisations are selected to optimise latency for simple queries while maintaining quality for complex ones.

**What This Version Delivers:**

## Features

### OPTIMISE-003: Automatic Model Routing (30-40 hours)

**Routing Strategy:**
- Simple factual → `llama3.2:3b` (fast)
- Complex analytical → `llama3.2:70b` (quality)
- Creative → `mistral:latest` (balanced)
- Vision → `llava` (multi-modal)

**Success Criteria:**
- [ ] 30-50% latency reduction for simple queries
- [ ] Routing accuracy >90%
- [ ] Model selection appropriate per query type
