# Version Design Plans

**Purpose:** High-level design plans for each version

---

## Overview

This directory contains version-specific design overviews that outline the features, goals, and scope for each ragged version.

**Distinction:**
- **planning/version/** = High-level design goals (what)
- **roadmap/version/** = Detailed implementation plans (how + when)
- **implementation/version/** = What was actually built

---

## Versions

### Long-Term Strategic Planning (v1.0+)

**Executive Summary:** [Strategic Roadmap](../../vision/strategic-roadmap.md) - Complete v1-v3 vision (~2600-3460 hours)

- **[v1.0/](./v1.0/)** - Personal Knowledge Platform (400-500h)
  - State-of-the-art PKM with modern web UI
  - Block-based editor, command palette, graph visualisation
  - PWA features, daily notes, template system

- **[v2.0/](./v2.0/)** - Enterprise & Compliance (270-340h)
  - Multi-user collaboration and real-time editing
  - GDPR/HIPAA/SOC 2 compliance frameworks
  - Self-hosted deployment for privacy-conscious organisations

- **[v3.0/](./v3.0/)** - Ecosystem Platform (850-1250h, modular)
  - Plugin marketplace and community extensions
  - Federated sharing across trusted instances
  - AI agents and mobile applications

### Foundation Releases (v0.x - In Progress)

- **[v0.5/](./v0.5/)** - Advanced memory and personas
- **[v0.4/](./v0.4/)** - Knowledge graphs
- **[v0.3/](./v0.3/)** - Advanced retrieval and multi-modal
- **[v0.2/](./v0.2/)** - Enhanced retrieval (current)
- **v0.1/** - MVP (complete - see implementation/)
