# Requirements Documentation

**Purpose:** User needs and functional requirements

---

## Overview

This directory contains user stories and functional requirements organised by feature area and persona.

---

## Structure

- **user-stories/** - User stories organised by feature and persona

---

## User Story Format

Each user story follows the template:
```
As a [persona],
I want to [action],
So that [benefit].
```

With acceptance criteria, priority, and effort estimates.

---

## For Contributors

**Adding a new user story:**
1. Identify the persona (researcher, developer, etc.)
2. Use the user story template
3. Link to related architecture/design docs
4. Assign priority and estimate effort (in hours)

**Templates:**
- See `../../process/templates/user-story-template.md`
