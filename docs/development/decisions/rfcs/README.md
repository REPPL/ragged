# Request for Comments (RFCs)

**Purpose:** Major feature proposals seeking community feedback.

---

## Overview

This directory will contain Request for Comments (RFCs) for substantial changes that need discussion before implementation.

## When to Create an RFC

- **Major features** that affect architecture or user experience
- **Breaking changes** that require community input
- **Technical design discussions** for complex features
- **NOT for:** Bug fixes or minor improvements

## RFC Format

Each RFC should include:

1. **Title and Summary** - What is being proposed
2. **Motivation** - Why this change is needed
3. **Design** - Technical design and architecture
4. **Alternatives** - What alternatives were considered
5. **Trade-offs** - Benefits and drawbacks
6. **Implementation** - Timeline and approach
7. **Community Feedback** - Discussion and questions

## File Naming

RFCs are numbered sequentially:
```
NNNN-feature-name.md
```

Example: `0001-multi-modal-rag.md`
