# Contributing to ragged

Welcome! We're excited you're interested in contributing to ragged.

---

## Current Status

**ragged is currently in the planning/early development phase** (pre-v0.1).

We will **welcome contributions after v0.1 is implemented** and the basic architecture is stable.

## Contributing After v0.1

Once v0.1 is released, we'll welcome contributions in these areas:

### Code Contributions

**Types of contributions**:
- Bug fixes
- Feature implementations (from roadmap)
- Performance improvements
- Test coverage improvements
- Documentation improvements

**Process** (will be detailed post-v0.1):
1. Check existing issues and discussions
2. Create an issue to discuss your planned contribution
3. Fork the repository
4. Create a feature branch
5. Write code + tests + documentation
6. Submit pull request
7. Address review feedback

### Documentation Contributions

**Types**:
- Tutorials for new features
- How-to guides for common tasks
- Explanation articles about RAG concepts
- API reference improvements
- Typo fixes, clarifications

**Process**:
- Same as code contributions
- Documentation-only PRs are welcome!

### Community Contributions

**Ways to help**:
- Answer questions in GitHub Discussions
- Help triage issues
- Test pre-release versions
- Share ragged with others
- Write blog posts about using ragged

## Development Setup (After v0.1)

*Will be documented once v0.1 is released.*

Planned setup will include:
- Python 3.10+ virtual environment
- Development dependencies
- Pre-commit hooks
- Testing framework setup
- Local Ollama installation

## Code Standards (Planned)

We will follow:
- **PEP 8** via `black` formatter (--skip-string-normalisation)
- **Type hints** checked with `mypy` (strict mode)
- **Linting** with `ruff`
- **Testing** with `pytest` (>80% coverage)
- **Docstrings** in Google style
- **British English** in all documentation

## AI-Assisted Development Policy

**Disclosure Required**:
- If you use AI tools (GitHub Copilot, Claude Code, etc.) in your contribution, please disclose this in your PR
- Indicate which parts used AI assistance
- Confirm you understand and can explain all submitted code

**Why**: Helps maintainers assess review effort and ensures quality

See [AI Assistance Guidelines](../development/methodology/ai-assistance.md) for details.
